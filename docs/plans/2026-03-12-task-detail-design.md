# 任务详情页设计说明

## 1. 目标

本次实现目标是在现有任务监控页与失败任务重试能力基础上，补齐“任务详情基础可读页”：

- 从任务列表进入任务详情页
- 展示任务基础字段、状态、时间信息
- 展示完整错误信息
- 提供基础返回入口

本轮重点是把任务页从列表观察，推进到单任务可读，而不是一次性做复杂状态流转页。

## 2. 方案选择

本次采用“独立任务详情页”方案。

选择原因如下：

- 和当前知识库创建 / 编辑 / 详情等页面风格一致
- 后续扩展时间线、步骤明细或更多治理动作更自然
- 比抽屉或弹窗更适合承载完整错误信息和更多字段

## 3. 页面结构

新增路由 `/tasks/:id`。

任务监控页操作列新增 `详情` 入口。

详情页分为两部分：

### 3.1 顶部概览区

展示：

- `taskId`
- `taskType`
- `taskStatus`
- `bizId`
- `createdAt`
- `updatedAt`

### 3.2 详细信息区

展示：

- `errorMessage`
- 其他基础字段
- 返回任务列表入口

## 4. 数据流设计

详情页对接 `GET /api/admin/tasks/{taskId}`。

页面状态处理如下：

- 加载中：骨架屏
- 加载失败：页面级错误态
- 加载成功：展示概览和详细信息

## 5. 业务关联策略

本轮对业务关联采取保守策略：

- 若只有 `bizId`，则先展示业务 ID
- 不在前端强行猜测业务类型
- 只有在后端字段足够明确时，后续才补业务跳转入口

## 6. 本轮边界

本轮包含：

- 任务详情页
- 任务列表详情入口
- 基础字段展示
- 完整错误信息展示
- 加载态与失败态

本轮不包含：

- 状态流转时间线
- 任务步骤明细
- 详情页内重试操作
- 强依赖 `bizType` 的业务跳转

## 7. 验证范围

本轮至少完成以下验证：

- 可从任务列表进入详情页
- 详情页可成功加载任务详情
- 骨架态、失败态、正常态清晰可区分
- `errorMessage` 可完整展示
- 前端项目可通过 `npm run build`

## 8. 后续建议

在本轮完成后，建议继续按以下顺序推进：

1. 任务状态流转时间线
2. 失败原因增强展示
3. 任务与文档 / 知识库详情联动
