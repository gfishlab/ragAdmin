# 知识库模型边界实施计划

## 实施范围

本轮实现围绕“知识库只绑定向量模型、聊天模型统一走平台默认、取消默认向量模型兜底”展开，包含后端契约、前端表单、迁移脚本、文档与测试同步。

## 任务拆分

### 1. 后端知识库契约收口

- 移除知识库请求与响应中的 `chatModelId`
- 将 `embeddingModelId` 改为必填
- 更新 `KnowledgeBaseService` 的创建、更新、详情与列表映射逻辑

### 2. 运行时模型解析收口

- 调整 `ChatService`，知识库场景统一走平台默认聊天模型
- 调整 `AppChatService`，前台聊天模型优先级改为“请求 > 会话 > 平台默认”
- 调整 `ModelService`，移除默认向量模型解析与相关引用校验
- 调整 `ModelBootstrapInitializer`，不再根据配置文件补录默认向量模型

### 3. 前端知识库页面同步

- 移除知识库表单中的聊天模型选择
- 向量模型改为必选
- 列表页、详情页与辅助文案同步到新规则
- 向量索引页补充“未绑定”状态展示

### 4. 数据与文档同步

- 新增 Flyway 迁移，约束未来知识库写入必须包含 `embedding_model_id`
- 更新架构文档、API 文档、SQL 草案、验收文档与实施计划

### 5. 验证与收口

- 更新模型服务、问答服务、WebMvc 层相关测试
- 执行后端定向测试与前端构建验证
- 完成提交与推送
