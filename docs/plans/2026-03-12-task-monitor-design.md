# 任务监控页设计说明

## 1. 目标

本次实现目标是在现有知识库、文档上传、文档解析触发闭环基础上，补齐“任务可观察性基础页”：

- 提供独立任务监控页面
- 支持任务分页列表
- 支持最小筛选条件
- 支持刷新与重置
- 让文档解析等异步流程有一个可查看的入口

本轮重点是建立任务可观察性，而不是一次性做成完整任务中心。

## 2. 方案选择

本次采用“独立任务页 + 基础筛选 + 刷新”的方案。

选择原因如下：

- 信息架构清晰，符合当前后台导航结构
- 能以最小成本提供任务可观察性
- 不会因为提前引入详情、重试、轮询等能力，导致本轮实现失焦

## 3. 页面结构

新增独立页面 `/tasks`，左侧导航中新增入口“任务监控”。

页面由两部分构成：

### 3.1 筛选区

首版筛选项如下：

- `taskStatus`
- `taskType`
- `bizId`
- 刷新按钮
- 重置按钮

### 3.2 列表区

列表首版展示字段如下：

- `taskId`
- `taskType`
- `taskStatus`
- `bizId`
- `errorMessage`
- `createdAt`
- `updatedAt`

## 4. 数据流设计

任务页对接 `GET /api/admin/tasks`。

查询参数使用：

- `taskType`
- `taskStatus`
- `bizId`
- `pageNo`
- `pageSize`

分页与筛选联动规则：

- 首次进入页面，按默认分页加载
- 变更筛选条件后，重置到第一页
- 点击刷新时，保留当前筛选和当前页
- 点击重置时，恢复默认筛选并回到第一页

## 5. 错误与状态处理

状态展示规则如下：

- `WAITING` / `RUNNING` 使用警示色
- `SUCCESS` 使用成功色
- `FAILED` / `CANCELED` 使用危险或灰色

错误处理规则如下：

- 列表加载失败：展示列表区错误态与重试入口
- 空结果：展示清晰空态
- 正常结果：展示表格与分页

## 6. 本轮边界

本轮包含：

- 导航入口
- 独立任务页
- 基础筛选
- 分页列表
- 刷新与重置
- 状态颜色区分

本轮不包含：

- 任务详情页
- 失败任务重试
- 自动轮询
- 任务统计图表
- 高级筛选能力

## 7. 验证范围

本轮至少完成以下验证：

- 可从左侧导航进入任务页
- 任务列表可按筛选条件刷新
- 分页、刷新、重置行为正确
- 列表失败、空态、正常态可明确区分
- 前端项目可通过 `npm run build`

## 8. 后续建议

在本轮完成后，建议继续按以下顺序推进：

1. 任务详情页
2. 失败任务重试
3. 任务与文档详情联动
