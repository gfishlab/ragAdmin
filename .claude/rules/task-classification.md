# 任务分类规则

## 目标

在任务开始前识别任务类型、影响范围和默认加载对象，让后续规则和记忆以渐进式方式进入上下文。

## 一级分类

- 后端实现
- 前端实现
- 数据库 / Schema
- 文档 / 规范
- 架构设计
- 流程机制
- 多模块联动

## 分类判断

- 涉及服务端实现时，归为后端实现
- 涉及页面、交互、前端状态管理时，归为前端实现
- 涉及 SQL 草案、Flyway、表结构、索引、向量维度时，归为数据库 / Schema
- 涉及入口文件、运行目录、协作流程、复盘机制时，归为流程机制
- 同时影响多个模块边界时，归为多模块联动

## 分类后的默认动作

- 后端实现：优先核对架构文档、API 文档、包结构规则与相关工程记忆
- 前端实现：优先核对页面边界、接口契约、异步流程约定与相关工程记忆
- 数据库 / Schema：优先核对 SQL 草案、命名一致性、迁移方式与相关工程记忆
- 文档 / 规范：优先核对术语一致性、事实来源与阶段文档归档方式
- 流程机制：优先核对顶层入口文件与运行目录的职责边界
- 多模块联动：优先启用 `planner` 形成拆解与验收边界
